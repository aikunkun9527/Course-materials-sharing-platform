# 课程资料分享平台 - 架构设计文档

## 一、项目概述

### 1.1 项目背景
开发一个基于云计算的课程资料分享平台，允许学生注册、登录、上传和下载课程资料，并为每门课程提供讨论区功能。

### 1.2 核心功能
- 用户注册与登录
- 课程资料上传/下载
- 课程讨论区
- 用户权限管理

### 1.3 技术栈要求
- **服务器**: 阿里云轻量应用服务器
- **关系型数据库**: 阿里云 RDS MySQL
- **文件存储**: 阿里云 对象存储 (OSS)

### 2.2 三层架构说明

#### 2.2.1 表现层 (Presentation Layer)
- **技术**: React.js
- **职责**:
  - 用户界面展示
  - 用户交互处理
  - HTTP请求发送
  - 响应数据渲染

#### 2.2.2 应用层 (Application Layer)
- **技术**: Node.js (Express)
- **部署**: 阿里云轻量应用服务器 + Docker
- **职责**:
  - 业务逻辑处理
  - 请求路由分发
  - 数据验证
  - 身份认证与授权
  - 调用数据库和文件存储服务

#### 2.2.3 数据层 (Data Layer)
- **关系型数据**: 阿里云 RDS MySQL
  - 存储结构化数据（用户、课程、资料、讨论等）
- **文件存储**: 阿里云 OSS
  - 存储非结构化数据（PDF、PPT、视频、图片等）

## 三、技术选型方案

### 3.1 前端技术栈

| 技术 | 说明 | 版本建议 |
|------|------|---------|
| React | 前端框架 | 18+ |
| Axios | HTTP客户端 | 1.6+ |
| React Router | 路由管理 | 6+ |
| Ant Design | UI组件库 | 5+ |
| Vite | 构建工具 | 5+ |
| Zustand | 状态管理 | 4+ |

**推荐选择**: React 18 + Ant Design + Vite
- 生态成熟，组件丰富
- 开发效率高
- 社区活跃

### 3.2 后端技术栈

| 技术 | 说明 | 版本建议 |
|------|------|---------|
| Node.js | 运行时环境 | 18+ |
| Express | Web框架 | 4+ |
| JWT | 身份认证 | jsonwebtoken 9+ |
| MySQL2 | 数据库驱动 | 3+ |
| Multer | 文件上传处理 | 1.4+ |
| OSS SDK | 阿里云OSS SDK | ali-oss 6+ |
| Bcrypt | 密码加密 | bcryptjs 2+ |

**推荐选择**: Node.js 18 + Express + MySQL2
- JavaScript全栈，技术栈统一
- 异步性能优异
- 生态丰富

### 3.3 云服务选型

#### 3.3.1 应用部署 - 阿里云轻量应用服务器
- **优势**:
  - 成本低(¥60-100/月)
  - 易于维护
  - Docker容器化部署
  - 完全控制权
  - 适合中小型应用

#### 3.3.2 关系型数据库 - 阿里云 RDS MySQL
- **优势**:
  - 100%兼容MySQL
  - 性能优异，支持海量数据
  - 高可用（主备架构）
  - 自动备份
  - 按需付费

#### 3.3.3 文件存储 - 阿里云 OSS
- **优势**:
  - 海量存储，成本低
  - 高可靠（99.9999999999%）
  - 支持CDN加速
  - 提供临时URL授权访问
  - 基本免费额度

## 四、系统功能模块

### 4.1 用户模块
- 用户注册（邮箱）
- 用户登录（JWT认证）
- 密码加密存储（bcrypt）
- 个人信息管理
- 头像上传

### 4.2 课程模块
- 课程浏览/搜索
- 课程创建（所有用户）
- 课程分类管理
- 课程成员管理
- 我的课程（创建+加入的聚合）

### 4.3 资料模块
- 资料上传（支持多种格式）
- 资料下载
- 资料预览（PDF、图片）
- 资料分类/标签
- 资料搜索
- 上传者信息展示
- 下载次数统计

### 4.4 讨论模块
- 课程讨论区（每门课程独立）
- 发布讨论主题
- 评论/回复
- 点赞功能
- 非成员可浏览内容

### 4.5 权限模块
- 基于角色的访问控制（RBAC）
- 角色：学生、管理员
- 所有用户都可以创建课程
- 资料访问权限控制

## 五、安全设计

### 5.1 身份认证
- 使用JWT (JSON Web Token)进行无状态认证
- Token过期时间设置
- RefreshToken机制

### 5.2 数据安全
- 密码使用bcrypt加密（salt rounds: 10）
- SQL注入防护（使用参数化查询）
- XSS防护（前端输入转义）
- CSRF防护（Token验证）

### 5.3 文件安全
- 文件类型白名单验证
- 文件大小限制
- OSS签名URL过期时间控制

### 5.4 网络安全
- 强制HTTPS通信
- CORS配置
- 请求频率限制（Rate Limiting）
- API访问日志记录

## 七、开发与部署流程

### 7.1 开发环境
- 本地开发：Docker Compose搭建开发环境
- 本地数据库：MySQL 8.0
- 本地文件存储：模拟OSS（本地文件系统）

### 7.2 部署流程
1. **代码托管**: GitHub
2. **服务器准备**: 轻量应用服务器
3. **应用部署**: Docker容器部署
4. **数据库部署**: RDS MySQL创建实例
5. **OSS配置**: 创建Bucket，配置访问权限

### 7.3 部署步骤简述
```bash
# 1. 连接服务器
ssh root@server-ip

# 2. 克隆项目
git clone https://github.com/aikunkun9527/Course-materials-sharing-platform.git

# 3. 配置环境变量
cd course-platform
cp .env.production.template .env.production
vim .env.production

# 4. 构建Docker镜像
docker build -t course-platform-backend:latest ./backend
docker build -t course-platform-frontend:latest ./frontend

# 5. 启动服务
docker-compose up -d

# 6. 配置Nginx
cp nginx-course-platform.conf /etc/nginx/sites-available/course-platform
systemctl restart nginx
```
